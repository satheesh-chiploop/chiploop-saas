# Co-simulation runner steps

## Prerequisites

Install and verify the following tools:

- Verilator
- Python 3
- pytest
- cocotb
- Cargo / Rust toolchain
- Optional coverage tools:
  - llvm-cov
  - Verilator coverage support

Suggested verification commands:

- `verilator --version`
- `python3 --version`
- `pytest --version`
- `cargo --version`

## Environment

Set or override the following environment variables before running:

- `RTL_TOP` : top-level RTL module name
- `RTL_DIR` : RTL source directory
- `FILELIST` : RTL file list
- `FW_DIR` : firmware workspace directory, if different from repository root
- `COSIM_OUT` : output directory for co-simulation artifacts

Recommended defaults:

- `FW_DIR=${FW_DIR:-.}`
- `COSIM_OUT=${COSIM_OUT:-firmware/validate/out}`

## Build RTL simulation

Preferred flow:
1. If `firmware/validate/verilator_build.md` exists, follow the command sequence defined there.
2. Otherwise build a Verilator simulation using the imported RTL sources and file list.

Example command sequence:

- `verilator -cc --exe --build --top-module "${RTL_TOP}" -Mdir "${COSIM_OUT}/verilator_obj" -CFLAGS "-O2" -f "${FILELIST}"`

If the project uses a custom wrapper or additional include paths, supply them via the Verilator build instructions or environment variables documented by the imported package.

## Build firmware

If the firmware crate is present and intended for the co-sim path:

- `cargo build --manifest-path "${FW_DIR}/Cargo.toml"`

If the firmware layer is library-only or the repository provides a different manifest location, use that manifest path instead. For validation runs, a debug build is usually sufficient.

## Run cocotb

Run the cocotb tests through pytest and direct outputs under `firmware/validate/`:

- `pytest -q firmware/validate --junitxml=firmware/validate/out/junit.xml`

If the cocotb test harness requires a Verilator binary or shared object generated by a separate build step, use the generated artifact paths from the Verilator build output and the repository-provided test configuration.

## Expected outputs

After a successful run, expect the following artifacts under `firmware/validate/out/` or the configured output directory:

- simulation log(s)
- pytest output
- optional JUnit XML report
- Verilator build logs
- optional coverage artifacts:
  - RTL coverage database or report from Verilator coverage
  - firmware coverage report from llvm-cov, if enabled and wired into the test flow

Typical output examples:

- `firmware/validate/out/junit.xml`
- `firmware/validate/out/verilator_build.log`
- `firmware/validate/out/cosim.log`
- `firmware/validate/out/coverage/`
